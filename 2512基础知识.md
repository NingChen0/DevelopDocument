### 计算机硬件

##### 多线程

> 软件并发模型， 运行一个进程同时执行多个线程， 在单核CPU 上操作系统可以
>
> 通过时间片轮转的方式实现多线程的并发

多核： 一个CPU 包含多个处理核心，每个核心独立执行指令-->并行处理多任务

超线程： 现代CPU让每个物理核心模拟出两个逻辑核心，从而同时运行更多的线程

一个cpu同一时刻只能处理一个进程，所以可能是为并行

##### 磁盘

* 盘片：一个硬盘包含多个`盘片`，每个盘片有上下`两个盘面`
* 每个`磁头`可以读取一点换新区域，盘面上以`同心圆`形式划分的环形区域。从外圈到内圈，一圈一圈排列，称为`磁道`
* 将一个戈丁手臂位置上所有的磁道合起来，多个盘片上相同半径的磁道组成一个“柱面”，形成一个`柱面`
* 每个磁道又被划分为若干个弧段，称为`扇区`，扇区典型的值是512字节

> 硬盘使用磁化方向来表示二进制位，磁化方向翻转代表 “1”，无翻转代表 “0”
>
> 数据存放于一段一段的扇区内，通过改变盘面磁性颗粒的磁化方向，从而“写入”二进制数据

##### 虚拟内存

* 原理： 将正在使用的程序放入内存中执行，暂时不需要执行的程序放到磁盘的某个地方，这块地方称为虚拟内存，对应linux的swap， 当物理内存紧张时，内核会将不活跃的内存页写入 swap，腾出物理内存给更急需的进程

##### I/O设备

* I/O设备包含俩个部分： `设备控制器`和`设备本身`

  `设备控制器（硬件）`： 是物理芯片或者电路板，通常集成在主板，或者扩展卡（网卡，显卡），直接连接并控制具体的设备硬件

  * 作用： 接收来自CPU的命令，控制设备执行具体操作（如移动磁头，启动电机）

  `设备驱动程序（软件）`： 运行在操作系统内核空间的软件代码

  * 作用： 为操作系统提供统一接口，用来与特定型号的设备控制器通信

    将操作系统的I/O请求翻译成设备控制器能理解的具体寄存器或者命令序列

  🔄 **驱动程序 ↔ 控制器 ↔ 设备** 形成一个完整的控制链

  ​		两者如何协作？（以读取硬盘为例）
  用户程序调用 read(fd, buf, size)；
  操作系统文件系统层将请求转为“读 LBA 扇区 X”；
  驱动程序被调用，它知道如何与 SATA 控制器通信；
  驱动程序向 SATA 控制器的寄存器写入命令和参数；
  设备控制器接收命令，控制硬盘马达、磁头，从盘片读取数据；
  数据通过 DMA 直接传入内存；
  控制器发出中断；
  驱动程序响应中断，通知操作系统“读取完成”；
  用户程序得到数据。

#### 计算机启动流程

启动流程

1. 计算机加电

2. BIOS开始运行，检测硬件：cpu、内存、硬盘等

3. BIOS读取CMOS存储器中的参数，选择启动设备

4. 从启动设备上读取第一个扇区的内容（MBR主引导记录512字节，前446为引导信息，后64为分区信息，最后两个为标志位）

5. 根据分区信息读入bootloader启动装载模块，启动操作系统

6. 然后操作系统询问BIOS，以获得配置信息。对于每种设备，系统会检查其设备驱动程序是否存在，如果没有，系统则会要求用户按照设备驱动程序。一旦有了全部的设备驱动程序，操作系统就将它们调入内核。然后初始有关的表格（如进程表），穿件需要的进程，并在每个终端上启动登录程序或GUI

#### DHCP 原理

客户端和DHCP服务器四步骤交互（DORA）

> Discovery-- Offer-- request -- acknowledgment

1. Discover

   客户端开机连接网络发现本机没有ip

   客户端广播出去一个DHCP Discover 报文（目标mac :`ff:ff:ff:ff:ff:ff`）对应IP `255.255.255.255`

   源IP地址`0.0.0.0 `(尚未拥有IP)

2. offer 

   一个或多个DHCP收到 客户端发送的Discover后，从地址池中预留一个可用IP

   DHCP 服务器向客户端单播或者广播发送DHCP offer报文（提供的IP，子网掩码，租期，网关和DNS）

   客户端只接受第一个DHCP Offer报文

3. Request

   客户端收到第一个DHCP offer 时，，广播一个DHCP Request 报文， 声明这个客户端要用这个IP了。通知其他的DHCP 服务器不用他们提供的IP，请释放这个预留IP

4. DHCP Acknowledge( 确认)

   被选中的DHCP 服务器接收到Request 后，正式确认分配，发送DHCP ACK 报文

   客户端收到ACK后，正式使用该IP地址，并开始计时租期

   如果服务器无法分配，则发送DHCP NAK（否定应答）， 客户端重启发送Discover请求

#### DNS 域名解析

1. dns 的两种查询方式

   * 递归查询

     客户端发起请求后，本地DNS查不到IP，本地DNS服务器必须代劳到底， 本地服务器自己去完成后续（可能用迭代的方式访问根故服务器、顶级域名服务器等）的所有查询，把最终的IP地址返回给客户端

   * 迭代查询

     客户端发起请求后，本地服务器如未查到，去访问根服务器，跟服务器没有查到，会返回另一个可能知道的DNS服务器地址。

     本地DNS服务器与其他DNS服务器之间使用迭代查询

   客户端浏览器请求访问流程

2. DNS缓存

   DNS返回正确的IP之后，系统会把这个结果临时存储起来。并设定一个失效时间，在这个失效时间之内再次访问这个网站时，系统会直接从电脑本地的DNS缓存中把结果交还给你，而不必再次去查询DNS服务器了

3. 浏览器DNS查找顺序

   浏览器DNS缓存-->本地系统DNS缓存--> 本地计算hosts文件--> ISP DNS 缓存--> DNS服务器递归or 迭代查询

### 特殊的IP地址

广播地址： 用于向网络中的所有设备进行广播， 具有正常的网络号部分，，主机号部分全为1（即255）,

回环地址：127.0.0.1 系统作为诊断使用

私有地址： 只能局域网内使用，不能在internet上使用

* 10.0.0.0-10.255.255.255 （A类地址）
* 172.16.0.0-172.16.255.255（B类地址）
* 192.168.0.0-192.168.255.255（C类地址）

0.0.0.0 表示所有不清楚主机（本机路由表没有特定条目指明如何到达）和目的网络

##### linux 目录

1. `/etc/passwd` 存储用户账号基本信息，

   ```shell
   _chrony:x:129:135:Chrony daemon,,,:/var/lib/chrony:/usr/sbin/nologin
   user2:x:1001:1001::/home/user2:/bin/sh
   user3:x:1002:1002:,,,:/home/user3:/bin/bash
   # 用户名：密码占位符：用户ID:组ID：用户描述：用户家目录：登录的shell
   ```

2. `/etc/shadow` 安全存储用户密码，是对 /etc/passwd 的补充，目的是将敏感的密码信息从公开可读的 /etc/passwd 中分离出来

   ```shell
   user2:!:20445:0:99999:7:::
   user3:$6$zDnlcLur.Kq3r98D$NBN1S/a0mpnelpfaVL0gDzAdCMDhtzk/kCbyr72wjtOluud01MAjNaREdoC0TPw6MYBZ5NMIPLAuOsXS.ohy31:20445:0:99999:7:::
   # 9个字段，用冒号分离
   # 用户名:加密密码:上次修改密码的日期:最小密码年龄:最大密码年龄:警告期:非活动宽限期:账户过期日期:保留字段
   ```

3. `/etc/group` 定义用户组（groups） 及其成员的核心配置文件，它与 `/etc/passwd `和 `/etc/shadow `一起，构成了系统用户和权限管理的基础

   ```shell
   cdrom:x:24:user
   floppy:x:25:
   tape:x:26:
   sudo:x:27:user
   # 组名:组密码占位符:GID:组成员列表
   ```
