### 2.进程管理--系统负载高排障

* 排查流程

  1. 通过监控软件发现系统负载高

  2. 判断是cpu还是IO导致的负载高

     cpu高：top中的us(user用户占用cpu)  sy（system系统占用cpu）

     IO高： top 中的wa(iowait)磁盘io导致负载高

  3. 如果是cpu导致的，排查出哪个进程导致的  ps aux cpu使用率
  4. 如果是io导致的，排查初级哪个进程导致的（iotop）命令

### 4. 定时任务

* 定时任务案例： 同步时间，备份，通过脚本获取系统指标，通过邮件发送出去

* crond守护进程读取定时任务列表，使用crontab来编辑自己的定时任务

* corntab用于对定时任务的规则进行增删改查

  ```shell
  crontab -l #list 列出当前用户的crontab任务
  crontab -e #edit 编辑当前用户的定时任务
  crontab -r #remove 删除当前用户的定时任务
  ```

  ```shell
  vi /var/spool/cron/root # ==crontab -e #root=当前用户名文件夹
  cat /var/spool/cron/root #==crontab -l 
  ```

* 一个典型的crontab定时任务书写格式，文件条目：

  ```shell
  #规则：什么时间做什么事情
  # 分    时     日   月   星期    命令
  #0-59  0-23  1-31 1-12  0-6    command
   * * * * * command to be executed
   #eg:
  30 08 * * * go to school # 每天8点30分去学校
  #注意
  00 00 * * * to do something # 每天晚上12点执行
  * 00 * * * to do something #每天晚上12点00-59分钟都会执行一次
  ```

* 定时人物的特殊符号

  | 事件部分特殊符号 | 说明              | 案例                                                         |
  | ---------------- | ----------------- | ------------------------------------------------------------ |
  | /                | 每隔xxx时间       | */2 * * * * 每两分钟                                         |
  | -                | 表示范围          | 00 08-22 * * * 8点到22点执行，00 08-20/3 * * * 8-20点之间每三小时运行 |
  | ，               | 表示独立时间      | 00 08,11,16 * * * 在8点，11点，16点运行                      |
  | *                | 每，全部/所有时间 | * 在分钟上指示0-59                                           |

* ```shell
  #每天上午七点到晚上十一点内每两个小时运行cmd命令
  00 07-23/2 * * * CMD
  # 每天23点到第二天7点运行
  00 23,00-07 * * * CMD
  ```

* **环境变量故障案例**

  > 书写定时任务时，需使用却对路径
  >
  > 在定时任务运行命令或脚本时，只能识别到/bin或/usr/bin目录下的命令

  * 故障案例

    ```shell
    #现象：命令行执行命令或脚本时是成功的，交到定时任务中执行提示命令找不到
    #原因:定时任务执行脚本时，只能识别到/bin或/usr/bin下的命令
    #解决方案：
    	1在脚本中书写命令的绝对路径
    	2在脚本开头，重新定义PATH环境变量
    ```

    ```shell
    #重新定义PATH环境变量
    #!/bin/bash
    export PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin
    #...脚本代码
    ...
    ```

* 故障案例2
  
    ```shell
    #现象：邮件服务未开启，不断收到邮箱 you have new mail in /var/spool/mail/root
  #邮件服务关闭，一些邮件的临时目录不断堆积小文件 /var/spool/postfix/maildrop

    ```
  
  * 案例3
  
    定时备份/etc/目录到/backup以ip地址命名的目录中
  
    ```shell
    #1.取出ip地址存放到变量 2.取出时间存放到变量 3. 创建以IP命名目录
    # 4. 打包压缩存放到这个ip地址命名的目录中，压缩包加上时间
    #!/bin/bash
    # backup /etc/*  
    # author:ning
    ip=`hostname -I | awk '{print $1}'`
    time=`date +%F_%w`
    dir_name=/backup/
    
    mkdir -p ${backup_dir}$ip
    tar -zcvf /${dir_name}${ip}//etc-${time}.tar.gz /etc/
    ```
