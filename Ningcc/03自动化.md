* --rm : 自动删除容器，当容器停止时自动删除，一般用于测试环境
* --restart： `自动重启`--restart=always

:heavy_exclamation_mark:`exec` 和attach的区别

| docker container 指令 | 共同点     | 区别                                                         |
| --------------------- | ---------- | ------------------------------------------------------------ |
| exec                  | 连接到容器 | 连接的时候创建一个新终端（新的进程），<br />不通的exec互不影响 |
| attach                | 连接到终端 | 1. 容器要有终端，只能连接到已有的终端<br />共用一个终端，所有链接都一样<br />不会创建新的进程，而是直接连接到容器的现有进程 |

#### 4. 数据卷(共享目录)

> Docker 数据卷是用于容器与主机之间共享数据的工具。它们允许数据在容器之间或容器与主机之间持久化和共享，而不依赖于容器的生命周期。以下是关于 Docker 数据卷的一些关键点：
>
> 1. 创建和管理
> 创建数据卷：可以使用 docker volume create 命令来创建一个新的数据卷。
> 查看数据卷：使用 docker volume ls 来列出当前系统上的所有数据卷。
> 检查数据卷：通过 docker volume inspect 可以获取某个特定数据卷的详细信息

在启动容器时，可以通过 -v 或 --mount 标志将数据卷挂载到容器内。例如：

```shell
docker run -d -v mydata:/app nginx
#这会创建一个名为 mydata 的数据卷，并将其挂载到容器内的 /app 目录下。
```

##### 02. Dockerfile中的指令

·`都是大写`

| 指令   | 含义                                                   | 应用                                                         | 建议                                                   |
| ------ | ------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------ |
| FROM   | 指定基本镜像：==docker pull xxx                        | FROM ubuntu：20.04                                           | 尽量使用具体版本                                       |
| LABLEL | 指定容器的具体信息，author，xx                         |                                                              |                                                        |
| ENV    | 用于创建dockerfile中使用的变量                         | ENV Tengine_version 2.3.3                                    | 软件版本可使用变量                                     |
| RUN    | 制作镜像过程中需要执行的命令                           | RUN 系统命令即可                                             | 不建议使用连续多个RUN，合并多个RUN                     |
| ADD    | 可以把指定文件拷贝到容器中（指定目录） ` 会解压压缩包` | ADD restart.tar.gz /app/code/restart/                        | 拷贝压缩包使用                                         |
| COPY   | 可以把自定文件目录拷贝到容器中，不支持自动解压         | COPY nginx.conf /etc/ningx/nginx.conf                        | 拷贝文件或目录                                         |
| EXPOSE | 指定镜像要对外暴露的端口                               | EXPOSE 80                                                    | 用于指定一个或多个容器的端口，未来这个端口可以被-p识别 |

vi ~/mysql/master/conf/my.cnf

```bash
[mysqld]
# 服务器唯一id，默认1
server-id=1
# 是否只读，1为只读,0为读写,
read-only=0
# 设置日志格式，默认ROW
binlog_format=ROW # 日志格式，ROW格式通常更适用于复制
#二进制日志名称，默认binlog
log-bin=binlog
# 需要复制的数据库
binlog-do-db=mytest
# 设置不需要复制的数据库
#binlog-ignore-db=mysql
#binlog-ignore-db=information_schema
expire_logs_days=7 # 二进制日志自动清理的天数

```
