# 使用 VMware 虚拟机实现子网络划分与互通



> ***\*实验时间\****：2025年11月  

> ***\*实验平台\****：VMware Workstation + Windows/Linux 虚拟机  

> ***\*目标\****：在单台物理主机上构建两个隔离私有子网，通过路由器虚拟机实现跨子网通信，同时保留各虚拟机访问互联网的能力。



---



## 一、实验目标



- 创建两个独立私有子网：

- 子网 A：`172.16.10.0/24`

- 子网 B：`172.16.70.0/24`

- 每台客户机（PC1、PC2）配置双网卡：

- 网卡1：NAT 模式（用于访问外网）

* 网卡2：连接私有子网（用于内网通信）

- 部署一台双网卡 ***\*Router 虚拟机\****，实现子网间路由转发

- 实现效果：

- ✅ PC1 与 PC2 可互相 ping 通

 \- ✅ 各自仍能正常访问互联网



---



## 二、网络拓扑

| 设备   | 网卡 1（NAT）    | 网卡 2（私有子网）     | IP 地址（私有）   |

|----------|--------------------|--------------------------|------------------|

| ***\*PC1\****  | VMnet8（自动获取） | VMnet2          | `172.16.10.3`   |

| ***\*PC2\****  | VMnet8（自动获取） | VMnet3          | `172.16.70.2`   |

| ***\*Router\**** | —         | VMnet2 + VMnet3      | `172.16.10.254`<br>`172.16.70.254` |



> 所有私有网络均使用 ***\*VMware 自定义 Host-only 网络\****，关闭 DHCP，手动分配 IP。



---



## 三、实施步骤



### 1. 配置 VMware 虚拟网络



1. 打开 ***\*编辑 → 虚拟网络编辑器\****

2. 添加两个自定义网络：

- ***\*VMnet2\****：子网 `172.16.10.0`，掩码 `255.255.255.0`，模式：仅主机，***\*关闭 DHCP\****

- ***\*VMnet3\****：子网 `172.16.70.0`，掩码 `255.255.255.0`，模式：仅主机，***\*关闭 DHCP\****



### 2. 为虚拟机添加第二张网卡



- 关闭虚拟机

- 编辑设置 → 添加网络适配器 → 自定义 → 选择对应 VMnet

- PC1 → VMnet2

- PC2 → VMnet3

- Router → 添加两张网卡，分别连 VMnet2 和 VMnet3



### 3. 配置 IP 与路由



#### Router（推荐 Linux）

```bash
# 配置 IP

sudo ip addr add 172.16.10.254/24 dev eth0

sudo ip addr add 172.16.70.254/24 dev eth1



# 启用 IP 转发

echo 'net.ipv4.ip_forward=1' | sudo tee -a /etc/sysctl.conf

sudo sysctl -p
```
